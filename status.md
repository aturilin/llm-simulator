## Дизайн-принципы

- **Show don't tell** — сначала визуал, текст по клику
- **Трансформация данных** — каждый экран: "было" (узнаваемое) → процесс → "стало"
- **Единый стиль раскрытия** — `›` скобка слева, поворачивается при открытии
- **Цвета токенов** — розовый, оранжевый, циан, зелёный, accent, жёлтый, красный — переиспользуются
- **Подсказки** — tooltip по клику (info-click, pill-q), показывается над элементом
- **Контраст масштаба** — 4 vs 2000, 1 ноутбук vs 375
- **Breadcrumbs убраны** — step indicator вверху достаточно
- **Скролл вместо уменьшения** — ноутбуки/параметры одного размера, страница скроллится

## Технические детали

- Один файл: `index.html` (inline CSS + JS)
- 12 слайдов, горизонтальные переходы, стрелки ← →
- Slides sorted by data-step (DOM order ≠ display order)
- Tooltip вынесен на уровень body (z-index:300), lazy lookup по ID
- Короткий контент — align-items:center, длинный — flex-start + overflow scroll
- scrollTop = 0 при переходе на новый слайд
- Attention auto-plays при переходе на экран 7
- Training auto-plays при переходе на экран 9

## Ключевые решения (из обсуждений)

- **Порядок экранов**: Ввод → System Prompt → API (сначала добавляется harness, потом отправляется)
- **Токенизация**: BPE режет по статистике, не по грамматике
- **Эмбеддинги**: условные названия измерений для наглядности, не буквальные
- **Attention**: каждый токен обновляет свой вектор, впитывая контекст от релевантных
- **FFN**: 2 слоя (4096→16384→4096), простая структура, магия в масштабе
- **MoE**: в pill-подсказке, не отдельный экран (перебор для чайника)
- **Нет буквального "Париж" в векторе**: вектор сдвигается в абстрактном направлении, при декодировании ведущем к токену "Пар"
